# PostgreSQL Setup for Railway - Manual Guide

## Step 1: Create PostgreSQL on Railway (Via Dashboard)

1. Go to: https://railway.com/project/57b1b937-ce99-4533-a4a7-4281f0108dbf

2. Click **"+ New Service"** or **"Add Service"**

3. Select **"PostgreSQL"** from the list

4. Click **"Create"** and Railway will auto-provision a PostgreSQL instance

## Step 2: Get Connection String

Once PostgreSQL is created:

1. Click on the **"PostgreSQL"** service in your Railway dashboard

2. Go to **"Variables"** tab

3. You'll see `DATABASE_URL` (auto-generated by Railway)

4. Copy the full connection string (looks like):
   ```
   postgresql://user:password@host:port/railway
   ```

## Step 3: Add DATABASE_URL to Your Backend Service

1. Click on your **"Express Entry"** service (the Python backend)

2. Go to **"Variables"** tab

3. Click **"Add Variable"**

4. Key: `DATABASE_URL`
   Value: Paste the PostgreSQL connection string from Step 2

5. Click **"Save"**

## Step 4: Redeploy

Once DATABASE_URL is added:

1. Go back to the **"Express Entry"** service

2. Click **"Redeploy Latest"** or **"Trigger Deploy"**

3. Wait 2-5 minutes for it to build and start

4. Check **"Logs"** tab to see if it connects successfully

## Testing the Connection:

Once deployed, you can test by visiting:
```
https://your-railway-url.onrender.com/api/health
```

You should get:
```json
{"status": "ok"}
```

If you get a database error, the DATABASE_URL wasn't set correctly.

## Troubleshooting:

### Error: "could not translate host name"
- DATABASE_URL is wrong
- Copy it again from Railway PostgreSQL variables

### Error: "password authentication failed"
- Username/password in connection string is wrong
- Railway auto-generates them, just copy as-is

### Error: "does not exist" for tables
- Means connection worked but tables weren't created
- Check the app logs for migration errors
